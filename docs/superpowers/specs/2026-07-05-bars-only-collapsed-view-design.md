# 折りたたみ時バーのみ表示 + 手動更新ボタン 設計書

日付: 2026-07-05
対象: `claude_usage_widget.py`

## 目的

1. 折りたたみ(展開前)時のウィジェットを大幅に低くするため、文字を一切表示せず
   消費量バーだけの1行にする。
2. 追加クレジット(extra_usage)を金額文字ではなくバーで表示する。月間制限
   (`monthly_limit`)を100%とする。
3. 手動更新ボタン(⟳)を追加する。連打されても取得処理が多重実行されないようにする。

## 折りたたみ時のレイアウト

```
┌────────────────────────────────┐
│ ████░░░ ███░░░░ █░░░░░░  ⟳ ▼  │
└────────────────────────────────┘
   5h      週     追加クレジット
```

- バー3本: 5時間 / 週間 / 追加クレジット。各 Canvas 幅80px・高さ8px。
- ラベル・%・金額・区切り文字「│」は表示しない(「残 $X」「API $X」ラベルは削除)。
- 右端に ⟳(手動更新)と ▼(展開)の記号アイコンのみ。フォントは FONT_SMALL。
- 行の余白は pady=2 に縮小(現行4)。ツールチップは付けない。
- APIプリペイド残高は展開後の詳細パネルのみに表示(現行の詳細行を維持)。
  折りたたみ行の `prepaid_label` は削除し、`refresh_prepaid()` は詳細パネルの
  再描画のみを担う形に整理する。

## 追加クレジットバーの仕様

- 塗り率 = `extra_used / extra_limit × 100`(使用率。月間制限=100%)。
- 色は既存 `bar_color()` の閾値(80%警告 / 95%危険)を流用。severity は None を渡す。
- N/A 条件(グレーの空バー、%表示なし): `extra_enabled` が偽、`extra_used` または
  `extra_limit` が None、`extra_limit <= 0` のいずれか。
- 使用率の計算は純粋関数 `extra_percent(snap) -> float | None` として切り出し、
  単体テスト可能にする。
- stale(取得失敗時)は他バーと同様にグレー化する(`_mark_stale` に追加)。

## 手動更新ボタン(⟳)と連打対策

- ⟳ クリックで `Poller.poll_now()` を呼ぶ(即時取得 + 次回タイマー再スケジュール。
  トレイの「今すぐ更新」と同じ経路)。
- `Poller` に `_in_flight` フラグを追加する:
  - `poll_now()` は先に `reschedule()` し、`_in_flight` が真なら **スレッドを起動せず
    return**(タイマー経路・トレイ経路・⟳連打のすべてで多重fetchを防ぐ)。
  - スレッド起動直前に `_in_flight = True`。ワーカー `_fetch_bg()` 全体を
    try/finally で包み、finally で必ず `_in_flight = False` に戻す
    (早期return・例外でも解除漏れなし)。
  - フラグの読み書きはUIスレッド(セット)とワーカースレッド(クリア)からだが、
    単一bool代入のためGIL下で安全。取りこぼしても次回クリックで再試行可能。
- 実行中の視覚フィードバック: 取得中は ⟳ をグレー(FG_DIM より暗い BAR_BG 色)に
  して完了で戻す。`Poller` から `widget.set_refreshing(bool)` を `root.after` 経由で
  呼ぶ。busy中のクリックは無視される(poll_nowが即return)。

## 変更しない点

- エラー時のステータス行(文字)は現行どおり折りたたみ時も表示する。
  「文字なし」は通常時のみの方針。
- 展開後の詳細パネル(各リミット%・リセット時刻・追加クレジット金額・
  APIプリペイド・更新間隔)は現行のまま。
- ドラッグ移動・トレイ・ポーリング間隔設定の挙動。

## _Bar クラスの変更

- ラベル(`tk.Label`)と `pct_label` を削除し、Canvas のみの構成にする。
- 高さは定数 `BAR_HEIGHT = 8` とし、`update()` の塗り矩形もこれを使う。
- stale 引数の挙動は現行のまま。

## 改訂 (2026-07-06)

実装後のフィードバックにより、折りたたみ行をさらに低くするため以下を変更:

- 行の pady を 2 → 0 に。
- ⟳▼ を文字ラベルから `ICON_SIZE = 12` px の Canvas 描画アイコンに変更
  (▼=三角形ポリゴン、⟳=円弧+矢頭ポリゴン)。文字のフォント行高さ(約16px)が
  行の高さを支配していたため。busy 中は ⟳ を BAR_BG 色で描画。
  展開中は ▲(上向き三角形)を描画。
- 結果: 折りたたみ高さ 21px → 12px(実測)。

## 改訂2 (2026-07-06): レート制限(HTTP 429)の専用ハンドリング

運用開始後、usage API が短時間の連続リクエスト(自動ポーリング+⟳手動更新)で
HTTP 429 を返すことが判明(Retry-After ヘッダーは常に 0 で実用性なし)。
従来は他エラーと同じ「更新失敗」扱いでバーがグレー化し、実際より深刻に見えた。

- `RateLimitError(FetchError)` を追加し、`fetch_usage()` は 429 で これを送出。
- `Poller._fetch_once` は `except RateLimitError` を `except FetchError` より先に
  置き、`widget.set_rate_limited()` のみ通知(set_status は呼ばない=グレー化しない)。
- `set_rate_limited()` は `_rate_limited` フラグを立てて ⟳ を COLOR_WARN(アンバー)で
  再描画するだけ。バー・ステータス行・トレイ色には触れない(表示中の値は直前の
  正常値のため)。高さ・幅は不変。
- 復帰: 次の取得成功(`apply_snapshot`)または本物のエラー(`set_status` 文言あり)で
  フラグ解除。busy(取得中の暗色)表示はフラグより優先。

## テスト

- `extra_percent()` の単体テスト: 正常系(50%など)、`extra_enabled=False`、
  used/limit 欠損、`limit=0` で None。
- 連打対策: `threading.Thread` をモンキーパッチし、`_in_flight=True` 中の
  `poll_now()` がスレッドを起動しないこと、finally でフラグが戻ることを検証。
- 既存テスト(parse/config/fetch)は影響なし。
